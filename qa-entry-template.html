Q&A Entry Template & Workflow Guide

Purpose: This document shows you how to add new Q&As to your community pages after approving questions and receiving answers.

Quick Workflow Overview

  1. Question arrives via form → Goes to Google Sheets
  2. You review → Approve/reject/edit for clarity
  3. You decide who answers:
    • You answer it yourself, OR
    • Forward to relevant directory professional (see email templates below)
  4. Answer received → You copy template below
  5. Fill in template with question + answer + metadata
  6. Paste into qa-noticeboard.html (all questions go here)
  7. If it's exceptional: Also paste into qa-curated.html with featured badge
  8. Commit and push to publish
  9. If user provided email: Send notification (see email templates)

Basic Q&A Entry Template

Copy this HTML structure and fill in the placeholders:

<div class="qa-entry" data-tags="TAG1,TAG2">
    <div class="question">QUESTION TEXT HERE?</div>
    <div class="question-meta">Asked by ASKER NAME • DATE</div>

    <div class="answer">
        <div class="answer-label">Answer:</div>
        <div class="answer-text">
            <p>ANSWER PARAGRAPH 1</p>

            <p>ANSWER PARAGRAPH 2 (if needed)</p>

            <p>ANSWER PARAGRAPH 3 (if needed)</p>
        </div>
        <div class="answerer">— Answered by <a href="directory.html">PROFESSIONAL NAME</a>, TITLE, LOCATION</div>
    </div>

    <div class="tags">
        <span class="tag">TAG 1</span>
        <span class="tag">TAG 2</span>
    </div>
</div>
Where to paste: Add this inside the <div id="qa-container"> section of qa-noticeboard.html. Newest questions go at the top (right after the opening tag).

Field Guide

data-tags=""

Add relevant tags separated by commas for filtering. Available tags:

  • rammed-earth - Rammed earth specific
  • cob - Cob construction
  • adobe - Adobe/mud brick
  • ceb - Compressed earth blocks
  • straw-clay - Light straw clay
  • earthbag - Earthbag/superadobe
  • mixed - Mixed techniques or general earthen building
  • professional - Professional practice (permits, codes, business)

Example: data-tags="rammed-earth,professional"

Asker Name

  • If they provided name: Use it
  • If they left it blank: Use "Anonymous"
  • Never use email address as name

Date

Use format: Month Day, Year (e.g., "January 15, 2025")

Answerer Attribution

  • If you answered: — Answered by Nathan Gordin (no link)
  • If directory professional answered: Link their name to directory.html
  • Include their title and location for context

Tags (visual)

Choose 1-3 descriptive tags that help users understand the topic at a glance:

  • Rammed Earth
  • Cob
  • Adobe
  • Equipment
  • Climate
  • Permits & Codes
  • Troubleshooting
  • Materials
  • Techniques

Example 2: Multiple Answers

When multiple professionals answer the same question, add additional answer blocks:

<div class="qa-entry" data-tags="cob,climate">
    <div class="question">Can cob walls survive in humid tropical climates?</div>
    <div class="question-meta">Asked by Anonymous • January 20, 2025</div>

    <div class="answer">
        <div class="answer-label">Answer:</div>
        <div class="answer-text">
            <p>Yes, but with careful design. I've built cob structures in Kerala, India (very humid) that have performed well for 15+ years. The key is generous roof overhangs (minimum 1 meter) and proper foundation height to prevent splash-back.</p>

            <p>Also critical: Allow 6-12 months drying time before plastering. Rush this and you'll trap moisture inside the wall.</p>
        </div>
        <div class="answerer">— Answered by <a href="directory.html">Priya Sharma</a>, Natural Builder, India</div>
    </div>

    <div class="answer">
        <div class="answer-label">Answer:</div>
        <div class="answer-text">
            <p>Different perspective from Ecuador: I prefer stabilizing with small amounts of lime (5-8%) for tropical climates. Not traditional cob purists' approach, but significantly improves moisture resistance while maintaining breathability.</p>

            <p>The trade-off: Slightly less thermal mass, but better long-term durability in high-humidity environments.</p>
        </div>
        <div class="answerer">— Answered by <a href="directory.html">Carlos Rodriguez</a>, Architect, Ecuador</div>
    </div>

    <div class="tags">
        <span class="tag">Cob</span>
        <span class="tag">Climate</span>
        <span class="tag">Troubleshooting</span>
    </div>

Multiple answers = valuable! Different perspectives show there's no single "right" answer and help users understand trade-offs.

Example 3: You Answered

When you answer instead of forwarding to a professional:

<div class="qa-entry" data-tags="rammed-earth,mixed">
    <div class="question">How do I fix erosion on an existing rammed earth wall?</div>
    <div class="question-meta">Asked by Anonymous • January 18, 2025</div>

    <div class="answer">
        <div class="answer-label">Answer:</div>
        <div class="answer-text">
            <p>First, identify the cause: Is it splash-back from inadequate roof overhang? Wind-driven rain? Rising damp from poor foundation drainage? The fix depends on the root cause.</p>

            <p>For surface erosion: Clean the damaged area, spray with water to dampen (not saturate), and patch with matching soil mix, compacting in thin layers. The patch should blend seamlessly if soil composition matches.</p>

            <p>Prevention: Extend roof overhangs if possible, improve site drainage, and consider applying a breathable lime wash as sacrificial weather protection.</p>
        </div>
        <div class="answerer">— Answered by Nathan Gordin</div>
    </div>

    <div class="tags">
        <span class="tag">Rammed Earth</span>
        <span class="tag">Troubleshooting</span>
    </div>
</div>

Publishing Workflow

  1. Open qa-noticeboard.html
  2. Find the <div id="qa-container"> section
  3. Add your new entry at the TOP (newest first)
  4. Save file
  5. Test locally (check filtering works with your tags)
  6. If featuring: Repeat for qa-curated.html
  7. Commit: git add . && git commit -m "Add Q&A: [brief description]"
  8. Push: git push -u origin [your-branch]
  9. Send notification email if user provided email

Tips for Quality

  • Edit questions for clarity: Rephrase if needed to make searchable/useful for others
  • Break up long answers: Multiple paragraphs read better than walls of text
  • Add context in answerer attribution: "Rammed Earth Contractor, Portugal" tells users the answer's perspective
  • Use data-tags strategically: Tags enable filtering, so choose accurately
  • Encourage multiple answers: Reach out to 2-3 professionals if question is substantial
  • Credit fairly: Always attribute answers, link to directory when applicable
