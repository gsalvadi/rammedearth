Community Q&A System - Complete Workflow

Purpose: This document explains the entire process for managing the Community Q&A feature, from question submission to publication.

Quick Reference Files

  • ask-question.html - Public question submission form
  • qa-noticeboard.html - All approved Q&As (complete archive)
  • qa-curated.html - Best-of selection (featured Q&As)
  • qa-entry-template.html - HTML templates for adding Q&As
  • qa-email-templates.html - Email templates for all communications
  • qa-professional-guide.html - Secret guide for professionals (not linked publicly)
  • qa-workflow-documentation.html - This file

The Complete Workflow

1 User Submits Question

User fills out form at ask-question.html with:

  • Question text (required)
  • Name/nickname (optional, defaults to "Anonymous")
  • Email (optional, but gets them notified when answered)

Submission goes to Google Sheets via Apps Script webhook.

2 Review Question in Google Sheets

Check your Google Sheet for new questions. Evaluate:

  • Quality: Is it specific enough to answer usefully?
  • Scope: Is it general enough to help others, or too project-specific?
  • Duplication: Has this been answered already?
  • Clarity: Does it need rewording for searchability?
Edit for clarity: Rephrase questions to make them more useful for future readers. "What's the best tamper for small residential projects?" is better than "what tamper should I buy?"
3 Decide: Answer or Forward?

For approved questions, decide who should answer:

  • You answer: If it's general advice, outside professionals' specific expertise, or quick to answer
  • Forward to professional: If it matches someone's specialty/location and would showcase their expertise
  • Multiple professionals: For substantial questions where different perspectives add value
4a If You're Answering

Write your answer (2-4 paragraphs, 100-300 words). Focus on:

  • Practical, actionable advice
  • Specific examples or recommendations
  • Acknowledging nuance and trade-offs

Then skip to Step 5.

4b If Forwarding to Professional
  1. Choose professional from directory based on specialty/location
  2. Use email template from qa-email-templates.html:
    • "Requesting Answer (First Time)" if they haven't answered before
    • "Requesting Answer (Returning)" if they have
  3. Include professional guide link for first-timers: [your-site]/qa-professional-guide.html
  4. Wait for their response (usually 3-7 days)
  5. Send thank-you email when received
5 Format the Q&A Entry
  1. Open qa-entry-template.html
  2. Copy the basic template
  3. Fill in:
    • Question text (edited for clarity if needed)
    • Asker name or "Anonymous"
    • Date (format: "January 15, 2025")
    • Answer text (from you or professional)
    • Answerer attribution with link to directory if applicable
    • data-tags for filtering (e.g., "rammed-earth,professional")
    • Visual tags (1-3 descriptive tags)
6 Publish to Notice Board
  1. Open qa-noticeboard.html
  2. Find <div id="qa-container">
  3. Paste your formatted entry at the TOP (newest first)
  4. Save file
  5. Test locally:
    • Open in browser
    • Test tag filtering works
    • Test search finds your content
    • Check links work
7 Optional: Add to Curated Page

If the Q&A is exceptionally valuable:

  1. Copy the entire entry from notice board
  2. Add <span class="featured-badge">Featured</span> at top
  3. Optionally add curator note explaining why it's featured
  4. Paste into qa-curated.html
What makes a Q&A "featured"? Practical advice, specific recommendations, represents multiple perspectives, or addresses common confusion.
8 Commit and Push
  1. git add qa-noticeboard.html qa-curated.html (or just noticeboard if not featured)
  2. git commit -m "Add Q&A: [brief description of question]"
  3. git push -u origin [your-branch]
9 Notify Asker (If Email Provided)
  1. Use email template: "Notifying Asker (Answer Posted)" from qa-email-templates.html
  2. Send from noreply@yourdomain.com to look automated
  3. Include link to qa-noticeboard.html
  4. Mention answerer's name and credentials
Tip: If multiple professionals answered, use the "Multiple Answers Posted" template instead.
10 Mark as Complete in Sheets

Update your tracking sheet:

  • Mark question as "Published"
  • Note date published
  • Note who answered
  • Note if user was notified

Weekly Routine Suggestion

  1. Monday: Check Google Sheets for new questions
  2. Tuesday: Review/edit questions, decide who should answer, send requests to professionals
  3. Wednesday-Friday: Wait for professional responses, write your own answers
  4. Weekend: Format, publish, commit/push, notify askers
Time estimate: 1-2 hours per week for 3-5 questions. Scales linearly with volume.

Quality Guidelines

Good Questions to Approve

  • Specific and practical
  • Has broader applicability beyond one person's project
  • Hasn't been answered already
  • Shows genuine interest in learning

Questions to Decline or Redirect

  • Too project-specific (requires site visit or detailed consultation)
  • Already answered in archive (point them to existing answer)
  • Not about earthen building
  • Asking for free design work

Good Answers

  • Practical and specific (not vague generalities)
  • Share real-world experience
  • Acknowledge nuance and trade-offs
  • 100-300 words (2-4 paragraphs)
  • Honest about limits of knowledge

Handling Edge Cases

Question is borderline quality

Edit it to make it better. Example:

  • Original: "how do i fix my wall"
  • Edited: "How do I fix erosion on an existing rammed earth wall?"

Professional declines to answer

Thank them, ask someone else or answer yourself. No pressure.

Professional's answer is low quality

Email back asking for more detail: "This is helpfulâ€”could you add a bit more about [specific aspect]? A couple more sentences would make it more useful for readers."

Duplicate question arrives

If already answered: Reply to asker with link to existing answer.
If not yet answered but in queue: Combine with existing question, note both askers.

No professionals specialize in that topic

Answer it yourself, or mark as "needs specialist" and revisit when you add relevant professionals to directory.

Question sparks Substack post idea

Perfect! Answer briefly in Q&A, then write expanded version for Substack. Link between them.

Setup Requirements (One-Time)

Google Apps Script

  1. Create Google Sheet for Q&A submissions
  2. Create Apps Script webhook (same as consultation forms)
  3. Update ask-question.html with your webhook URL

Email Setup

  • Configure noreply@yourdomain.com forwarding to your email (for manual sending)
  • Or set up email alias that looks automated

Professional Guide URL

  • Once site is live, update email templates with actual URL to qa-professional-guide.html

Success Metrics to Watch

  • Questions per week: Volume indicator
  • Professional participation: How many directory members answer questions
  • SEO traffic: Does Q&A bring in search traffic?
  • Conversion: Do Q&A readers book consultations or hire from directory?
  • Quality: Are answers getting multiple perspectives?
  • Themes: What topics come up repeatedly? (Fodder for Substack posts)

When to Evolve the System

If volume exceeds 10-15 questions/week:
  • Consider semi-automated publishing (Airtable + API)
  • Recruit volunteer moderators from directory professionals
  • Implement upvoting system to surface best answers
For now: Manual workflow is sustainable, gives you quality control, and lets you learn what questions actually come in. Don't over-engineer until you prove the demand.
